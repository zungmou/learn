这个故事是作者从他前同事的一段代码注释里挖出来的。需求很简单：从 `{1, 2, ..., n}` 里均匀随机地抽 `k` 个不重复的数。

代码长这样，短得离谱：

```python
def floyd(n, k):
    s = set()
    for i in range(n - k + 1, n + 1):
        t = random.randint(1, i)
        if t in s:
            s.add(i)
        else:
            s.add(t)
    return s
```

注意它只循环了 `k` 次，从 `n-k+1` 到 `n` 倒着走。每一步随机从 `[1..i]` 里抽一个 `t`，如果 `t` 已经被选过了，就把当前这个 `i` 放进结果；否则把 `t` 放进去。**分支逻辑就这么简单：撞上了就保底取 i，没撞上就取 t。**

作者说这算法比蓄水池采样（Reservoir Sampling）更难直观理解，因为那个分支让组合论证变得别扭。他给了两种理解视角：

---

### 视角一：增量构建

每一轮迭代，是把一个均匀分布在 `[1..n]` 上的 `k`-子集，加上一个从 `[1..n+1]` 抽的元素，映射成一个均匀分布在 `[1..n+1]` 上的 `(k+1)`-子集。

关键是证明这个映射是 **`(k+1)`-to-1** 的——每个可能的 `(k+1)`-子集恰好有 `k+1` 种输入能映射到它。两种情况分别论证，结论成立。

### 视角二：跟 Fisher-Yates 洗牌的关系

这个是作者的朋友 Paul 指出的。Fisher-Yates 洗牌有一种"向上"的变体：

```python
def shuffle(a):
    for i in range(0, len(a)):
        j = random.randint(0, i)
        a[i], a[j] = a[j], a[i]
```

每一步构建一个 `[0..i]` 上的均匀排列，然后扩展到 `[0..i+1]`。

关键是：**如果你对数组做完完整的 Fisher-Yates 乱序，只取最后 k 个元素，那恰好就是一个均匀随机子集。** 而弗洛伊德算法的本质，就是只模拟了那最后 k 次交换，而且还砍掉了实际挪动元素的开销，直接用 Set 做去重。

具体来说，把数组最后 `k` 个位置叫"尾部"。对于尾部中的每个原始元素 `i`：
- 如果它跟尾部里的 `j` 交换 → `i` 留在尾部，入选样本
- 如果它跟尾部外的 `j` 交换 → `j` 被换进尾部，入选样本

这两个分支，恰好对应了弗洛伊德算法的 `if t in s` / `else`。

---

所以这算法本质上就是 **Fisher-Yates 洗牌的"最后一公里"**——不从头洗完整个数组，只走最后那几步，然后用 Set 的 O(1) 查重来优雅地处理那些"谁该留在尾部"的判断。简洁、优雅、还带点反直觉的魔法感。
