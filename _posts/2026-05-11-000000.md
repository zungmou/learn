汉明距离

### 什么是汉明距离？

简单说就是：**两个等长的字符串，对应位置有多少个字符不一样，这个数量就是汉明距离。**

打个比方，你发了一条二进制消息 `1011101`，对方收到 `1001001`，逐位对比：

```
1011101
1001001
   ↑ ↑
```

第3位和第6位不一样，所以汉明距离就是 **2**。

再看几个例子：
- 数字串 `2143896` vs `2233796` → 第2、3、5位不同 → 距离 **3**
- 英文词 `"toned"` vs `"roses"` → 第1、2、4位不同 → 距离 **3**

### 跟异或的关系

如果是二进制串，还有个更酷的算法：把两个串做**异或（XOR）**，结果里有多少个 `1`，汉明距离就是多少。比如上面的例子：

```
1011101 XOR 1001001 = 0010100  → 两个1 → 距离2
```

### 谁提出的？干啥用的？

这玩意儿是以美国数学家 **理查德·汉明** 命名的。老爷子当年研究通信纠错码的时候提出了这个概念，所以汉明距离在：

- **信息论 / 编码理论** — 衡量纠错码的纠错能力，比如汉明码能纠正 1 位错误，就是因为它能保证任意两个合法码字之间的汉明距离至少是 3
- **密码学** — 分析加密算法的扩散效果
- **字符串相似度** — 判断两个等长字符串有多"像"

> ⚠️ 注意：汉明距离只适用于**等长**字符串。如果长度不同，就得用编辑距离（Levenshtein distance）那种更复杂的算法了。

### 一句话总结

**汉明距离 = 两个等长串之间"不一样的位置"有多少个，越少越相似。**
