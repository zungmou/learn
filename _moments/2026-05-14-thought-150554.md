[NASA 如何构建 Artemis II 的容错计算机](https://www.solidot.org/story?sid=84012)

**先感受一下时代的差距——**

阿波罗登月那会儿，计算机是个啥水平？1MHz 处理器、4KB 可擦除内存（注意是 KB，不是 GB），功能极其有限，很多环境和电源控制还是靠手动或者机电开关搞定的。搁现在就是个电子表级别的算力。

**而 Artemis II 的 Orion 飞船，堪称 NASA 史上容错最强的计算机系统，有多夸张呢？**

他们用了一种 **"叠甲"式架构**：
- 飞船上装了两台 **Vehicle Management Computers（飞船管理计算机）**
- 每台里面塞了 **两个 Flight Control Modules（飞控模块）**
- 所以一共 **4 个 FCM**，每个 FCM 又配了一对 **自检验处理器（self-checking processor pair）**
- 也就是说——**8 颗 CPU 同时并行跑飞行软件**

这还没完。太空里最怕的就是 **高能粒子辐射**，一个粒子打过来，CPU 里的 bit 可能就翻了个儿，运算就错了。他们的解法是：两个 CPU 组成一对互相校验，一旦哪个 CPU 因为辐射事件算错了，系统**立刻检测到并做出响应**。

**内存也是三重冗余**——每次读取数据，三份内存对比，自动纠正单比特错误。

**网络也是三重冗余**——所有交换机都带自检。

**最狠的一招：异构冗余**

飞船里还装了一套 **完全独立的 Backup Flight Software（BFS）备份飞行软件系统**。
- 不同的硬件
- 不同的操作系统
- 独立开发的、简化版飞行软件

意思就是：就算主系统集体抽风——比如同一个设计缺陷被辐射同时点爆——备份系统从根上就跟它不是一家人，不会犯一样的错。

**总结一下就是**：从 CPU 到内存到网络到软件，每一层都有冗余、有自检、有容错。太空不是开玩笑的地方，一颗粒子就可能让飞船变铁棺材，NASA 这是把"不怕你坏，就怕你坏了我不发现"做到了极致。
